## 🗺 Roadmap

앞으로 다룰(또는 여는) 트랙 — 관심 있는 주제가 있으면 issue 로 남겨주세요.

**Motion & Control** &nbsp;·&nbsp; `Local planning (DWA · Pure Pursuit · MPC)` &nbsp; `Trajectory optimization` &nbsp; `PID / LQR`

**Perception & State** &nbsp;·&nbsp; `SLAM` &nbsp; `Kalman / EKF / UKF` &nbsp; `Point cloud`

**Multi-agent** &nbsp;·&nbsp; `Prioritized A*` &nbsp; `CBS` &nbsp; `Swarm`

**Learning** &nbsp;·&nbsp; `RL for control` &nbsp; `Imitation learning`

## 🧭 How this org is organized

- **주제 하나 = 저장소 하나.** 스코프를 좁게 유지해 각 저장소가 그 주제의 이론·코드·실험을 자기완결적으로 담습니다.
- **구현 우선.** 라이브러리를 가져다 쓰기 전에 알고리즘을 밑바닥부터 구현하고, 같은 입력을 여러 언어로 대조합니다.
- **보이게 만든다.** 모든 알고리즘은 단계별 시각화(GIF/스냅샷)와 벤치마크로 검증합니다.
- **문서는 사이트로.** 정리가 충분히 쌓인 저장소는 `docs/` 를 정적 사이트(GitHub Pages)로 발행합니다.
